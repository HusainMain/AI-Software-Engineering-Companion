**Version:** 1.0

# Purpose

This document tracks products, tools, and platforms related to AI-assisted software development.

The goal is not to copy competitors, but to understand:

- What problems they solve.
- What problems remain unsolved.
- Where our product can create unique value.

This document will evolve continuously as new tools emerge.

---

# Research Methodology

Every competitor should be evaluated using the same framework.

For each product, document:

- Target audience
- Core functionality
- Strengths
- Weaknesses
- Pricing
- AI models used (if known)
- Local or cloud based
- Integration capabilities
- Problems solved
- Problems not solved
- Opportunities for differentiation

# Market Position

Current market observations:

- AI coding is becoming commoditized.
- Engineering decision support remains fragmented.
- Developers combine many tools to complete one project.
- Context and project memory remain major pain points.
- Security and architecture guidance are inconsistent.
- Tool discovery is still largely manual.

---

# Our Position

We are **not** building another AI coding assistant.

Instead, we aim to become an AI Software Engineering Companion that helps developers make better engineering decisions throughout the software development lifecycle.

Our product should integrate with existing coding assistants instead of replacing them.

---

# Differentiation Principles

Our product should differentiate itself by focusing on:

- Planning before coding.
- Engineering decision support.
- Long-term project memory.
- Context intelligence.
- Tool recommendations.
- Security guidance.
- Architecture reviews.
- Cost optimization.
- Software engineering education.
