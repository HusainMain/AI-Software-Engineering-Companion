# P-001 — Starting Without a Clear Project Plan

## Status

Validated

## Pain Score

⭐⭐⭐⭐⭐ (5/5)

## Frequency

Very High

## Confidence

98%

## Description

Many developers immediately ask an AI to generate code without first defining project goals, requirements, constraints, success criteria, or milestones.

As the project grows, this lack of planning leads to confusion, rewrites, feature creep, and inconsistent architecture.

## Symptoms

- Constantly changing direction
- Rewriting features
- Inconsistent implementation
- Confusing prompts
- Large context windows

## Root Cause

Developers underestimate the importance of planning because AI makes coding appear effortless.

## Existing Community Solutions

- Create a PRD first.
- Write a roadmap.
- Break work into milestones.
- Plan before prompting.

## Our Opportunity

Help developers generate and maintain structured project plans before coding begins.

# P-002 — Poor MVP Definition

## Status

Validated

## Pain Score

⭐⭐⭐⭐☆

## Frequency

High

## Confidence

95%

## Description

Developers often try to build complete products instead of identifying the smallest valuable version.

This increases complexity and delays validation.

## Symptoms

- Endless feature additions
- Large prompts
- Slow progress
- Difficulty finishing projects

## Root Cause

Lack of prioritization.

## Existing Community Solutions

- Define must-have features.
- Delay non-essential functionality.
- Build incrementally.

## Our Opportunity

Automatically identify and recommend a realistic MVP.

## Description

Developers continuously expand the project until it becomes difficult to maintain.

## Symptoms

- Growing backlog
- Inconsistent architecture
- Technical debt
- Reduced focus

## Root Cause

## Existing Community Solutions

- Maintain a roadmap.
- Prioritize features.
